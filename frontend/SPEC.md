# Socrates 高中数学教辅系统 - 前端演示系统设计规范

> **版本**: v1.0  
> **创建日期**: 2026-04-02  
> **用途**: 项目申报演示  
> **技术栈**: React 18 + TypeScript + Vite + TailwindCSS + Recharts

---

## 1. 设计理念

### 1.1 核心原则

1. **"看见AI思考过程" > "直接给答案"**：评委最想知道AI是怎么推理的，每一步都透明展示
2. **对比感**：学生方法 vs Canonical方法，同时展示
3. **数据讲故事**：用时间线、趋势图代替静态数据
4. **真实感 > 炫酷感**：Mock数据要有细节、有异常、有人情味

### 1.2 视觉风格

| 属性   | 选择                                                       |
| ------ | ---------------------------------------------------------- |
| 风格   | 科技感 + 教育亲和力，不是冰冷的工业风                      |
| 主色调 | `#2563EB`（蓝色，专业）+ `#10B981`（绿色，正确/进步）      |
| 强调色 | `#F59E0B`（黄色，警示/薄弱）+ `#EF4444`（红色，错误/危机） |
| 字体   | 系统默认（中文）+ `JetBrains Mono`（代码/数学公式）        |
| 动效   | 流畅但克制，每帧都有信息量                                 |
| 布局   | 大量留白，信息密度适中，评委2米外能看清                    |

### 1.3 Mock数据设计原则

```
所有Mock数据必须是"精心设计的真实感"：
- 有时间线：不是单点快照，是12天连续数据
- 有对比：干预前 vs 干预后，弱学生 vs 强学生
- 有异常：1-2个"需要关注"的学生，显示系统预警价值
- 有细节：具体数字，具体名字，具体错误类型
```

## 6. 技术实现

### 6.1 技术栈

```
React 18 + TypeScript + Vite
  ├─ TailwindCSS（样式，快速布局）
  ├─ Recharts（图表，数据可视化）
  ├─ React Router v6（路由）
  └─ Zustand（状态管理，用于Mock数据）
```

### 6.3 Mock数据注入策略

所有页面数据直接从 `src/mock/` 读取，不调用后端API。后端API接口通过注释标注，格式如下：

```typescript
// API: GET /api/students/:id
// 当前返回: MOCK_STUDENTS["student_001"]
// 真实实现: await apiClient.get(`/students/${id}`)

const student = useMockData("student", "student_001");
```
