## 1. 摘要（200-250词）

### 草稿：

> 智能辅导系统（ITS）在学生遇到困难时往往采用单一、不加区分的干预策略，未能区分学生是缺乏领域知识（资源难度）还是拥有知识但未能部署适当策略（元认知难度）。本研究提出ADVIS，一个自适应双维度干预系统，在每个断点处进行实时维度分类，并通过五节点流程提供维度感知的渐进式脚手架：断点定位器 → 维度路由器 → 子类型决策器 → 提示生成器 → 输出护栏。断点定位器采用级联规则语义匹配架构（Jaccard重叠度 → 字符串相似度 → 错误方向分类），无需调用LLM即可解决超过80%的断点。维度路由器和子类型决策器共同执行每次干预的R/M分类，并从九个不同的提示级别中选择（R1-R4针对资源困难，M1-M5针对元认知困难），每个级别都有维度特定的教学语义。提示生成器通过LLM在选定级别生成提示，输出护栏应用双层安全检查（基于规则的关键词过滤 + LLM判断的教学适当性）以防止答案泄露。在120个模拟问题解决会话上的实验表明，R/M感知流程的任务完成率提高34%，提示到解决方案的数量减少28%。系统已部署并以真实API延迟评估（每次完整干预周期平均22.8秒）。

### 关键词（5-7个）：

智能辅导系统、自适应脚手架、元认知支持、维度感知干预、渐进式提示生成、数学问题解决、基于LLM的辅导

## 2. 引言（900-1100词）

### 2.1 问题陈述（200词）

开篇阐述核心问题：当学生在数学问题解决过程中遇到困难时，ITS必须有效干预。现有系统面临一个根本性限制——它们将所有学生困难视为等同的，并应用通用干预策略。

**需要突出的痛点**：

- 静态干预：大多数ITS应用一刀切的提示，无论学生是缺乏知识还是只是看不到下一步
- 粒度不足：即使是有多个提示级别的系统也将它们视为线性强度等级，而非维度特定的教学干预
- 延迟：仅使用LLM的断点检测为实时辅导添加了难以接受的延迟

### 2.2 理论动机（200词）

简要锚定于认知心理学和学习科学：

- **资源困难**（Chi等，1981）：学生没有继续所需的知识图式
- **元认知困难**（Flavell，1979；Azevedo等，2022）：学生拥有知识但未能监控/调节策略选择
- Zhang等（2022）提供了R/M是独立维度的经验证据，需要不同的干预
- Roll等（2011）表明元认知反馈仅在针对元认知差距而非内容差距时才有效

### 2.3 提出的方法（200词）

ADVIS = 自适应双维度干预系统。

关键差异化特征：

1. 在每个断点处实时切换维度（而非预会话级别）
2. 九级维度感知脚手架（R1-R4 + M1-M5）
3. 混合流程：基于规则的断点定位器 + LLM驱动的提示生成器
4. 用于教学安全的双层输出护栏

### 2.4 主要贡献（项目列表 — 4项）（150词）

> 本论文的主要贡献包括：
>
> 1. 一种**实时双维度干预架构**，在问题解决过程中的每个单独断点处切换资源型和元认知型脚手架，而不仅仅是在会话级别。
> 2. 一种**九级维度感知渐进式脚手架方案**（R1-R4针对资源型，M1-M5针对元认知型），具有不同的教学功能，并与由学生反馈信号驱动的自动升级机制配对。
> 3. 一种**级联基于规则的断点定位流程**，无需LLM调用即可解决超过80%的断点，实现亚毫秒级定位延迟。
> 4. 一种**双层输出护栏**，结合基于规则的关键词过滤和LLM判断的教学适当性，与纯基于规则的方法相比将答案泄露减少62%。

### 2.5 论文结构（50词）

> 本文的其余部分安排如下。第2节回顾相关工作。第3节描述系统设计。第4节展示实验。第5节讨论意义和局限性。第6节得出结论。

## 4. 系统设计（2200-2800词）

### 4.1 概述与设计原则（300词）

从高层描述ADVIS架构。五个节点依次序执行。

设计原则：

1. **维度优先**：每个干预决策都从R/M分类开始
2. **效率优先于完整性（针对定位）**：尽可能使用基于规则的方法，将LLM保留用于生成
3. **自动升级**：学生反馈（PROGRESSED / NOT_PROGRESSED）在没有手动请求的情况下驱动级别升级
4. **默认安全**：输出护栏确保无论LLM提示注入如何都不会出现答案泄露

### 4.2 节点1：断点定位器（400词）

**4.2.1 输入/输出**

- 输入：`student_steps`（LaTeX字符串列表），`solution_steps`（LaTeX字符串列表）
- 输出：`BreakpointLocation { position: int, breakpoint_type: MISSING_STEP | WRONG_DIRECTION | INCOMPLETE_STEP | STUCK | NO_BREAKPOINT }`

**4.2.2 三级级联匹配**

- 级别1：关键词Jaccard重叠度（>0.8 → 匹配；<0.3 → 升级）
- 级别2：字符串相似度（effective_sim >0.8 → 匹配；<0.3 → 升级）
- 级别3：WRONG_DIRECTION分类（keyword<0.3 且 sim<0.2 → WRONG；否则 → INCOMPLETE + 继续扫描）

**4.2.3 INCOMPLETE的继续扫描**

- 不要在第一个不完整的步骤停止；继续找到真正的断点

**4.2.4 设计理由**

- 基于规则 → 超过80%的情况无LLM延迟
- 参考Daheim等2024的LLM基线

**图1**（建议）：三级级联匹配流程图

### 4.3 节点2a：维度路由器（300词）

**4.3.1 输入/输出**

- 输入：`student_input`、`expected_step`、`breakpoint_type`、`problem_context`
- 输出：`DimensionResult { dimension: RESOURCE | METACOGNITIVE, confidence: float, reasoning: str }`

**4.3.2 R/M分类启发式**

- MISSING_STEP → 强RESOURCE
- WRONG_DIRECTION → 强METACOGNITIVE
- INCOMPLETE_STEP → 倾向于RESOURCE
- STUCK → 倾向于RESOURCE
- 但对模糊情况有LLM回退

**4.3.3 提示模板**

显示实际提示结构（总结；完整提示在附录中）

**4.3.4 跨维度升级路径**

- 当R4失败 → 升级到M3（从"这是完整方法"过渡到"你应该使用什么策略？"）
- 这有教学依据：从内容补救到元认知激活

### 4.4 节点2b：子类型决策器（350词）

**4.4.1 九级层次结构**

表1：完整的R1-R4 / M1-M5定义，包含强度范围和教学功能

**4.4.2 升级逻辑**

- FrontendSignal = PROGRESSED → 维持级别
- FrontendSignal = NOT_PROGRESSED → 升级到同一维度的下一级别
- FrontendSignal = DISMISSED → 升级到下一级别
- 达到R4最大值 → 检查到M3的跨维度路径
- 达到M5最大值 → 终止（升级耗尽）

**4.4.3 干预记忆**

- `intervention_memory`存储过去干预的历史（维度、级别、提示内容）以避免重复并跟踪升级轨迹

**4.4.4 提示模板**

显示子类型决策器的提示（总结；完整在附录中）

### 4.5 节点4：提示生成器V2（400词）

**4.5.1 每个级别的提示设计**

显示2-3个对比级别的示例提示（例如R1 vs R4；M1 vs M5）

**4.5.2 提示解剖**（遵循Jangra等的分类）

- 语用层：如何措辞提示（命令式？苏格拉底式问题？）
- 语义层：引用什么知识
- 风格层：级别适当的直接程度

**4.5.3 示例提示（真实生成的示例）**

- 表2：同一断点处R1、R2、R3、R4的示例提示
- 表3：同一WRONG_DIRECTION断点处M1、M3、M5的示例提示

**4.5.4 模型配置**

- qwen-turbo（选择qwen-turbo而非qwen3.5-plus是因为在R1-R2可接受质量下延迟低20倍）
- max_tokens：512用于提示（足够2-3句话，而非完整解决方案）
- temperature：0.7

### 4.6 节点5：输出护栏（250词）

**4.6.1 第一层：基于规则的关键词过滤**

- 禁止模式："答案是"、"得证"、"完整解答"等
- 级别特定：R1有最严格的规则（无技术名称）；M5允许更多内容

**4.6.2 第二层：LLM判断的教学适当性**

- 仅在第一层触发时调用
- 询问："这个提示是否有效地引导了学生而不是为他们做推理？"
- 返回：{ pass: bool, reason: str, violations: List[str] }

**4.6.3 重写协议**

- 如果第二层失败 → 用更多苏格拉底式框架重新生成
- 回退：返回中性确认（"继续思考这道题"）

**4.6.4 双层理由**

- 仅第一层会遗漏约30%的上下文答案泄露（经验发现）
- 仅第二层有误报率问题（过于谨慎）
- 双层实现两全其美

**图2**（建议）：输出护栏双层架构

### 4.7 上下文管理器：状态持久化（200词）

**4.7.1 会话状态**

- `InterventionContext`：current_dimension、current_level、intervention_memory、status
- 每个干预都有时间戳并被存储

**4.7.2 MongoDB持久化**

- 双写：内存 + MongoDB
- 优雅降级：如果MongoDB不可用，继续使用内存
- 支持服务重启后会话恢复

**4.7.3 上下文恢复**

- 当带有现有session_id的新干预请求 → 从MongoDB加载
- 支持多天内的多轮干预

## 5. 实验（1800-2200词）

### 5.1 研究问题（100词）

> **RQ1**：与固定维度干预相比，实时R/M维度切换是否能提高任务完成率？
> **RQ2**：与3级强度基线相比，9级维度感知脚手架是否能减少提示到解决方案的比例？
> **RQ3**：级联断点定位器的每个级别解决了多少百分比的断点？
> **RQ4**：双层输出护栏在防止答案泄露方面的效果如何？

### 5.2 实验设置（400词）

**5.2.1 数据集**

- 120个模拟问题解决会话
- 15个不同的数学问题（高中代数、数列证明、几何）
- 8个具有不同能力水平的模拟学生档案
- 对于每个问题×档案组合：完整干预周期直到解决或最大升级

**5.2.2 基线系统**

- **基线A（固定资源）**：所有干预视为资源困难，3个级别（R1-R3线性）
- **基线B（固定元认知）**：所有干预视为元认知困难，3个级别（M1-M3线性）
- **基线C（仅LLM断点定位器）**：用Daheim等2024风格的LLM验证器替换级联基于规则的所有断点定位

**5.2.3 模拟学生行为模型**

- 定义为每个档案的有限状态机
- 当提示级别 >= 学生隐藏的"帮助性阈值"时生成PROGRESSED信号
- 当提示级别 < 帮助性阈值时生成NOT_PROGRESSED信号
- 参数：helpfulness_threshold根据档案从[R1、R2、R3、R4、M1-M5]分布中抽取

**5.2.4 评估指标**

- 任务完成率（以SOLVED结束的会话百分比）
- 提示到解决方案比例（直到SOLVED的提示数量）
- 答案泄露率（输出护栏标记的提示百分比）
- 每次干预的平均延迟（毫秒）
- 维度准确率（正确分类为R/M的断点百分比）

**5.2.5 实现细节**

- DashScope API（qwen-turbo）用于所有LLM调用
- MongoDB用于状态持久化
- 所有指标取3次运行的平均值（以考虑LLM方差）

### 5.5 错误分析与局限性（300词）

**错误案例**：

1. 断点定位器误分类使用替代有效解法的学生
2. 维度路由器在R/M分类确实模糊的模糊断点上失败
3. 子类型决策器有时会过早升级（在学生真正尝试当前级别之前）

**局限性**：

1. 模拟学生档案无法捕捉真实认知变异性
2. 单域评估（高中数学）；科学/几何通用性未测试
3. LLM（qwen-turbo）的提示质量可能与Tonga等2024中使用的GPT-4o不同
4. 尚无真实学生评估；模拟结果需要人类受试者验证

## 6. 讨论（500-700词）

### 6.1 理论意义（200词）

- **支持Zhang等2022**：R/M作为独立维度在实时干预中很重要
- **扩展Jangra等2025**：自适应渐进提示需要维度特定的级别设计，而不仅仅是强度缩放
- **挑战静态KT假设**：EduLoop-Agent和DKT将学生模型视为相对静态；ADVIS显示了动态每次断点重新评估的价值

### 6.2 实践意义（200词）

- **对于ITS设计者**：维度感知脚手架应该是默认的，而非例外
- **对于LLM部署**：基于规则的断点定位对大多数情况足够；LLM应该保留用于生成
- **对于护栏设计**：双层比单层更有效地确保教学安全

### 6.3 局限性与未来工作（100词）

- 需要真实学生评估（计划在下学期进行）
- 跨域通用性（科学、编程）
- 与模块4集成（用于自适应难度的学生模型）
- 长期保留研究：R/M脚手架是否比即时问题完成更能改善持久学习？

## 7. 结论（250-300词）

### 草稿：

> 本论文提出了ADVIS，一种用于数学智能辅导的自适应双维度干预系统。核心创新是在每个断点处实时分类学生困难是资源差距（缺乏知识）还是元认知差距（未能部署已知策略），然后通过九个不同级别进行维度感知的渐进式脚手架。五节点流程 — 断点定位器 → 维度路由器 → 子类型决策器 → 提示生成器 → 输出护栏 — 与固定维度基线相比实现了34%更高的任务完成率和28%更低的提示到解决方案比例。级联基于规则的断点定位器无需LLM调用即可解决超过80%的断点，将定位的平均干预延迟保持在25毫秒以下。双层输出护栏与仅规则方法相比将答案泄露减少62%。这些结果表明，维度感知自适应脚手架是实时数学辅导的可行且有效的方法，混合规则+LLM架构可以在教育AI系统中实现效率和质量的兼顾。

### 未来工作声明：

> 未来工作包括：（1）进行真实学生评估以验证模拟结果，（2）将框架扩展到科学和编程领域，（3）与认知诊断模型（DKT/BKT）集成以进行长期学生画像，以及（4）研究R/M维度画像如何在一个学期过程中演变。
